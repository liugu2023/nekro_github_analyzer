# GitHub 项目分析与评估

一个强大的 GitHub 项目综合分析工具，可以获取项目完整信息并进行质量评估。

## 功能概览

本插件提供两大类功能：

### 📊 项目信息分析
- 🔍 **项目基本信息**：名称、作者、描述、Star 数、Issues 数、Fork 数、贡献者数等
- 📖 **README 获取**：完整的项目文档内容
- 💾 **Commit 历史**：提交记录分析，最多获取 10000 条记录
- 📁 **项目结构**：目录和文件列表
- 🔀 **分支管理**：分支信息和保护策略
- 📦 **发行版本**：Release 和版本管理
- 🐛 **Issues & PRs**：问题跟踪和代码审查
- 🔎 **仓库搜索**：高级搜索功能，按 Stars、Forks 等排序

### ⭐ 项目质量评估
基于三大维度对项目进行综合评分（总分 0-100）：

1. **代码质量维度** (0-30 分)
   - 文档完整性 (10分)：README (5分)、LICENSE (2分)、CONTRIBUTING (1.5分)、CODE_OF_CONDUCT (1.5分)
   - 项目维护性 (10分)：Release 管理 (5分)、最后更新时间 (2.5分)、分支保护 (2.5分)
   - 代码组织 (10分)：主要语言 (3分)、语言数量合理性 (4分)、项目结构清晰度 (3分)

2. **项目活跃度维度** (0-40 分)
   - Release 发布频率 (12分)：过去一年的发行版本数量
   - 项目新鲜度 (15分)：基于最后更新时间（updated_at），衡量项目维护状态
   - Issue/PR 响应能力 (13分)：Issue 关闭率 (4.3分)、PR 合并率 (4.3分)、开放 Issue 健康度 (4.4分)

3. **社区健康度维度** (0-30 分)
   - 社区参与度 (15分)：Stars (6分，对数评分)、Forks (4分，对数评分)、贡献者数 (5分，对数评分)
   - 社区互动 (10分)：Issue 讨论活跃度 (5分)、PR 评审质量 (5分)
   - 项目成熟度 (5分)：项目年龄 (2分)、Star/Fork 比率 (2分)、持续维护 (1分)

**评级标准**：
- **A+ (85+)**：优秀 - 强烈推荐
- **A (75+)**：良好 - 推荐使用
- **B+ (65+)**：中上 - 可以使用
- **B (55+)**：中等 - 谨慎使用
- **C (45+)**：一般 - 需要评估
- **D (<45)**：较差 - 不推荐

## 支持的 URL 格式

```
https://github.com/owner/repo
https://github.com/owner/repo.git
git@github.com:owner/repo.git
owner/repo
```

## 配置选项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `GITHUB_TOKEN` | "" | GitHub Personal Access Token（可选） |
| `GITHUB_API_TIMEOUT` | 10 | API 请求超时时间（秒） |
| `EVALUATION_CACHE_TTL` | 1800 | 项目评分结果缓存时间（秒），范围 5 分钟到 24 小时 |
